# 第8章 系统实现

本章围绕系统落地过程，分别从核心功能代码实现、界面展示与关键业务逻辑三个层面展开说明。系统采用前后端分离架构：后端基于 Spring Boot 提供 RESTful 接口与业务服务，前端基于 Vue 3 构建多角色交互页面，最终形成“数据上报-风险识别-医生干预-平台治理”的完整运行闭环。

## 5.1 核心功能代码实现

### 5.1.1 登录认证与会话控制实现

系统认证采用 JWT 无状态方案。用户登录后由后端签发令牌，前端将令牌写入会话存储，并通过统一请求拦截器自动附加 Authorization 请求头。后端在过滤器中解析 token 后，完成身份恢复、角色绑定与登录版本校验。

1. 后端登录流程实现要点
- 用户名密码校验通过后，登录版本号加一并签发新 token。
- token 载荷包含 userId、role、loginVersion 等关键信息。
- 登录事件写入操作日志，便于后续审计。

2. 前端会话维护实现要点
- 路由守卫进入受保护页面前调用会话探活接口。
- 401 统一处理：清理会话并跳转登录页。
- 多标签页互斥登录：通过浏览器广播通道通知旧标签页下线。

3. 价值说明
该实现兼顾了安全性与可维护性，既能满足角色鉴权，又能实现同账号新登录顶掉旧登录的会话一致性要求。

### 5.1.2 健康数据上报与校验实现

健康数据模块是患者侧核心入口，主要实现数据采集、格式校验、分页查询、编辑删除与预警联动。

1. 上报实现机制
- 患者提交指标后，服务层按用户名反查 userId，保证数据归属准确。
- 保存数据后立即触发预警评估服务，实现实时风险识别。

2. 校验实现机制
- 血压采用 xx/xx 正则校验。
- 血糖、体重采用正数及范围校验。
- 服药指标仅允许已服药/未服药/1/0 四类值。
- 指标类型需处于启用状态，禁用指标禁止上报。

3. 查询与一致性机制
- 历史数据支持按指标类型、时间区间与分页参数检索。
- 对新增、修改、删除操作统一执行缓存失效，避免页面读取脏数据。

### 5.1.3 预警生成与医生闭环处理实现

系统预警能力由评估引擎与医生工作台协同完成。

1. 预警生成实现
- 上报后调用评估引擎，根据指标类型分派到对应 evaluator。
- 支持血压、血糖、体重三类规则判定。
- 输出 level、riskScore、riskLevel、reasonCode、reasonText 等风险要素。
- 对同一患者同类开放预警可执行更新复用，避免重复堆积。

2. 趋势能力实现
- 引入最近时间窗口数据做线性趋势预测。
- 在高风险或临界趋势场景下触发趋势型预警编码。
- 支持连续多天异常的主动预警增强。

3. 医生闭环处理实现
- 医生预警列表按可访问患者范围过滤，杜绝越权。
- 支持 riskLevel、minRiskScore、sortBy 等筛选排序参数。
- 处理预警时写入处理人、处理时间与备注，状态由 OPEN 转 CLOSED。

### 5.1.4 管理员治理能力实现

管理员模块承担平台治理职责，覆盖账号、规则、日志、反馈与监控。

1. 账号与权限治理
- 用户分页查询支持关键字、角色、状态筛选。
- 启停账号会联动登录版本更新，已登录会话即时失效。
- 角色权限支持查询与在线更新。

2. 配置治理
- 预警规则支持新增/更新，规则修改实时影响后续判定。
- 指标类型支持启停，形成前后台联动校验。
- 公告模块支持目标角色投放。

3. 审计与反馈治理
- 操作日志支持分页检索与 CSV 导出。
- 反馈模块支持单条处理、批量处理、按筛选条件批量流转与回复。
- 统计看板给出总量、待处理、近7日趋势等运行指标。

## 5.2 界面展示

系统界面按角色分为管理员端、医生端、患者端三条主线，并采用统一布局骨架，保障跨页面交互一致性。

### 5.2.1 登录与入口页展示

1. 登录页
- 展示系统品牌信息与账号密码输入区。
- 登录失败时显示明确错误信息。
- 登录成功后按角色跳转到工作首页。

2. 角色首页
- 管理员首页展示账号、规则、监控、日志、反馈等摘要卡片。
- 医生首页聚焦待处理预警、群组协作与反馈闭环。
- 患者首页聚焦上报、预警、反馈与健康提醒。

### 5.2.2 管理员端界面展示

1. 系统监控页
- 展示用户规模、上报规模、未处理预警等关键指标。
- 配套趋势与分布图表用于运行态分析。

2. 规则与配置页
- 预警规则、指标类型、系统公告集中管理。
- 支持启停与编辑，操作反馈清晰。

3. 审计与反馈页
- 日志页支持筛选、分页、导出。
- 反馈页支持状态流转、回复、批量处理。

### 5.2.3 医生端界面展示

1. 预警工作台
- 支持按风险等级和风险分筛选。
- 列表突出高风险对象与处理入口。

2. 群组管理页
- 支持群组创建、患者加入、医生协作成员维护。
- 支持查看群组患者与群组医生列表。

3. 患者洞察页
- 聚合患者档案、趋势数据、近期预警、未处理计数。
- 支持按指标类型与时间范围切换观察维度。

### 5.2.4 患者端界面展示

1. 健康上报与历史数据页
- 上报页支持多指标录入与即时反馈。
- 历史页支持按时间和指标筛选、编辑、删除。

2. 预警详情与阈值页
- 预警页展示风险等级、风险分与原因说明。
- 个性化阈值页支持患者自定义阈值并启停。

3. 周报月报页
- 汇总上报数量、预警数量、指标分布与风险趋势。
- 支持多图表展示，便于长期健康跟踪。

## 5.3 关键业务逻辑说明

### 5.3.1 鉴权与访问控制逻辑

1. 登录成功后签发 JWT，前端后续请求统一携带令牌。
2. JWT 过滤器校验 token、账号状态、登录版本一致性。
3. 安全配置按 URL 前缀进行角色隔离：
- /api/admin/** 仅管理员访问。
- /api/doctor/** 仅医生访问。
- /api/patient/** 仅患者访问。
- /api/feedback/** 允许患者与医生访问。
4. 未登录返回 401，越权访问返回 403。

### 5.3.2 数据隔离与越权防护逻辑

1. 患者侧数据隔离
- 患者档案、健康数据、预警查询均按当前登录用户绑定。
- 用户无法操作他人数据，越权请求直接拒绝。

2. 医生侧范围控制
- 医生预警查询先解析其可访问患者集合。
- 患者洞察与预警处理前再次校验医生对目标患者的可达性。
- 非团队患者访问或处理请求返回拒绝结果。

### 5.3.3 预警判定与风险分层逻辑

1. 规则读取优先级
- 若患者已启用个性化阈值，优先使用个性化规则。
- 否则回退到管理员配置的系统规则。

2. 判定与评分
- evaluator 根据指标类型计算风险等级与风险分。
- 风险等级采用 LOW/MEDIUM/HIGH 分层。
- 生成 reasonCode 与 reasonText 说明触发原因。

3. 趋势增强判定
- 使用近期历史数据识别持续上升趋势。
- 对可能超阈的情况提前生成趋势预警。

### 5.3.4 缓存与一致性逻辑

1. 高频查询能力
- 历史数据列表、预警列表、报表汇总、监控总览等场景采用缓存加速。

2. 一致性保障
- 对新增、修改、删除等写操作统一执行相关缓存失效。
- 通过“写后清缓存”策略避免旧数据长期驻留。

### 5.3.5 反馈闭环与审计可追溯逻辑

1. 反馈闭环
- 患者/医生提交反馈后进入待处理状态。
- 管理员处理并回复后，发送方可查看回复并标记已读。

2. 审计追溯
- 关键管理行为写入操作日志。
- 日志支持筛选、分页、导出，满足问题追踪与答辩证据留存。

## 本章小结

本章从代码实现、界面展示和关键业务逻辑三个维度说明了系统实现过程。系统不仅实现了基础功能可用，更通过会话安全、数据隔离、风险预警、反馈闭环与审计追溯等机制提升了工程完备度，能够支撑院外健康管理场景下的长期运行与持续演进。
