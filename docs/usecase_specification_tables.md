## 表2-1 用户登录用例规约表

| 项目 | 内容 |
| --- | --- |
| 用例编号 | 2-1 |
| 用例名称 | 用户登录 |
| 功能描述 | 用户提交账号和密码，系统完成身份认证并建立会话。 |
| 执行者 | 管理员、医生、患者 |
| 前置条件 | 用户账号已存在且状态为启用。 |
| 后置条件 | 系统签发令牌并记录登录日志，用户进入角色首页。 |
| 涉众利益 | 用户快速访问系统；平台确保登录安全与会话可控。 |
| 基本路径 | 1. 用户打开登录页面并输入账号和密码。2. 前端将凭据发送到认证服务。3. 认证服务校验账号是否存在、密码是否正确、账号状态是否启用。4. 认证服务更新登录版本并生成令牌。5. 认证服务返回用户信息和令牌。6. 前端保存会话信息并跳转工作首页。 |
| 扩展路径 | E1：账号或密码错误，系统返回认证失败提示。E2：账号被禁用，系统返回禁止登录提示。E3：网络异常，前端提示重试并保留输入。 |
| 字段列表 | username、password、token、userId、roleType、loginVersion、msg。 |
| 业务规则 | R1：账号密码必须同时输入。R2：仅启用账号可登录。R3：同账号新登录使旧会话失效。 |
| 备注 | 本用例是所有受保护业务用例的入口。 |

## 表2-5 风险预警生成用例规约表

| 项目 | 内容 |
| --- | --- |
| 用例编号 | 2-5 |
| 用例名称 | 风险预警生成 |
| 功能描述 | 系统在健康数据上报后自动完成规则判定与预警生成。 |
| 执行者 | 系统（预警引擎） |
| 前置条件 | 患者有新的健康上报记录；存在可用预警规则。 |
| 后置条件 | 产生或更新预警记录，并计算风险分与风险等级。 |
| 涉众利益 | 医生可快速识别重点风险患者；患者可及时收到风险反馈。 |
| 基本路径 | 1. 健康上报服务调用预警引擎。2. 预警引擎读取系统规则并尝试读取患者个性化阈值。3. 预警引擎按指标类型分派评估器。4. 评估器计算风险分、风险等级与触发原因。5. 系统检查是否存在同类开放预警。6. 若存在则更新原预警，若不存在则创建新预警。7. 系统返回评估完成状态。 |
| 扩展路径 | E1：无启用规则，系统不生成预警。E2：上报值不可解析，系统返回空判定并记录异常。E3：历史趋势满足条件，系统触发趋势类预警。 |
| 字段列表 | healthDataId、indicatorType、highRule、mediumRule、riskScore、riskLevel、reasonCode、status。 |
| 业务规则 | R1：个性化阈值优先于系统阈值。R2：风险分范围为0到100。R3：预警状态初始为OPEN。 |
| 备注 | 可映射为“上报服务-预警引擎-评估器-预警仓储”顺序图。 |

| 用例名称 | 医生预警处理 |
| 功能描述 | 医生按风险筛选预警并执行处理闭环。 |
| 执行者 | 医生 |
| 前置条件 | 医生已登录；医生与患者存在可访问关系。 |
| 后置条件 | 目标预警状态由OPEN变为CLOSED并记录处理信息。 |
| 涉众利益 | 医生提升干预效率；平台形成可追溯处置闭环。 |
| 基本路径 | 1. 医生进入预警工作台并设置风险筛选条件。2. 前端请求预警列表。3. 后端解析医生可访问患者集合并过滤预警。4. 系统返回排序后的开放预警列表。5. 医生选择目标预警并提交处理备注。6. 后端校验预警状态与患者可达性。7. 后端写入处理人、处理时间与备注并关闭预警。8. 前端刷新列表并展示处理成功。 |
| 扩展路径 | E1：预警已被处理，系统返回冲突提示。E2：医生越权处理，系统返回无权限提示。E3：筛选条件为空，系统返回默认排序结果。 |
| 字段列表 | riskLevel、minRiskScore、sortBy、alertId、handleRemark、handledBy、handledTime。 |
| 业务规则 | R1：医生仅可处理团队可访问患者预警。R2：仅OPEN状态允许处理。R3：默认按风险分降序。 |
| 备注 | 该用例直接体现权限模型与业务闭环。 |
