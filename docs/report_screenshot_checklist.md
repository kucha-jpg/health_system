# 答辩截图清单（系统实现章节）

日期：2026-04-07

## 1. 截图目标

用于支撑《系统实现》与《功能验证》章节，形成从登录到监控、预警、洞察、闭环的完整演示链路。

## 2. 建议截图顺序

1. 登录页品牌展示（含能力标签）
2. 角色首页（医生/患者/管理员入口）
3. 管理端监控总览（趋势+分布+群组+活跃用户）
4. 管理端预警规则管理
5. 医生预警工作台（高风险快捷视图）
6. 医生患者洞察页（趋势图+预警饼图+指标频次）
7. 患者周报/月报页（图表类型切换）
8. 患者预警列表与归档页
9. 系统公告与反馈管理
10. 审计日志页面（安全与可追溯性证据）

## 3. 每张图需保留的信息

1. 页面标题与副标题（体现业务语义）
2. 核心筛选条件（体现交互能力）
3. 关键数据区块（统计卡/图表/表格）
4. 操作结果反馈（成功提示、状态标签）

## 5. 演示讲解要点（单页30秒）

1. 这个页面解决什么问题。
2. 用户先做哪一步，再做哪一步。
3. 页面上哪块信息最能体现“智能预警/可视化/安全可追溯”。
4. 该页与前后页面如何形成闭环。

## 6. 交付建议

1. 截图统一 16:9 比例，保证答辩投影清晰。
2. 图表页面优先截“有数据且筛选已生效”的状态。
3. 关键截图附一行说明，可直接作为论文图注初稿。
