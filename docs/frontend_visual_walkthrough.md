# 前端视觉走查清单（管理员/医生/患者）

日期：2026-04-07

## 目标

对全站重设计进行按角色视觉走查，确保信息层级、交互入口与移动端表现一致。

## 已完成项

1. 全站主题切换能力已接入：支持森氧绿、海岸蓝、日光暖三套主题。
2. 顶栏操作区已支持换行，移动端不再出现按钮拥挤。
3. 全局补充 `fade-in-page` 动效类，已有页面标记可统一生效。
4. 主内容区增加 `min-width: 0`，修复个别页面在窄宽度下内容溢出风险。

## 管理员视角检查

- [x] 首页、监控、规则、日志、反馈页面有统一页头结构（标题+副标题+动作区）。
- [x] 表格页空状态统一可见（含插图和引导文案）。
- [x] 顶栏新增主题切换，不影响“处理反馈”快捷入口。

## 医生视角检查

- [x] 预警工作台保留高风险快捷视图和一键闭环入口。
- [x] 群组页/洞察页卡片、图表、表格视觉一致。
- [x] 移动端顶栏按钮与筛选控件不会重叠。

## 患者视角检查

- [x] 上报、历史、预警、周报、阈值页共享同一视觉骨架。
- [x] 阈值页与周报页在主题切换后颜色和对比度保持可读。
- [x] 登录/注册页风格保持独立品牌感，不与业务页冲突。

## 建议的下一轮微调（可选）

1. 为主题切换增加图标预览（当前为文本标签）。
2. 在关键图表页增加“高对比度模式”开关，方便投影演示。
3. 为患者端添加“紧凑布局”选项，提升小屏信息密度。
