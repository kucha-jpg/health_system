## 2. 调查研究法（问卷与访谈）

### 2.1 问卷对象与目标

- 对象：患者、医生、管理员（目标各 5-10 人）
- 目标：识别高频功能、痛点流程、预警接受度、可视化偏好

### 2.2 问卷题纲（可直接用于附录）

1. 您在院外健康管理中最常用的功能是哪些？
2. 您认为当前系统最耗时的操作步骤是什么？
3. 您对“异常预警”的可接受频率是多少？
4. 您更关注哪类图表（趋势/分布/对比/排名）？
5. 您是否需要按时间或指标筛选报表？
6. 您对账号安全（踢旧登录、审计日志）是否有明确需求？
7. 您认为系统还应增加哪些功能？

### 2.3 访谈提纲（半结构化）

- 患者：上报便捷性、预警可理解性、报告可读性
- 医生：群组协作效率、预警准确性、处理闭环体验
- 管理员：用户治理、规则维护、监控面板可用性

### 2.4 访谈结论（摘要）

- 患者更关注“上报简单 + 结果直观 + 及时提醒”。
- 医生更关注“高风险优先 + 低噪声预警 + 快速闭环”。
- 管理员更关注“可追溯审计 + 统一配置 + 多维监控”。

## 3. 案例分析法（对标维度）

对比维度：

- 角色划分是否清晰
- 院外连续数据追踪能力
- 预警策略与个性化能力
- 数据可视化与报表导出能力
- 安全机制与权限边界

系统对标结果：

- 具备三端角色闭环与群组协作能力。
- 已实现规则阈值 + 趋势预测 + 个性化阈值。
- 已实现多图表、多维统计、筛选与 CSV 导出。
- 已实现 JWT、细粒度授权、安全头、加密与审计日志。
