# 前端设计规范（统一视觉与交互）

日期：2026-04-07

## 1. 目标

本规范用于保证系统页面在视觉、交互、信息层次上的一致性，减少新增页面的风格漂移。

## 2. 页面骨架规范

所有业务页面建议采用统一结构：

1. `page-shell`：页面主容器
2. `page-header`：页头区（标题、副标题、操作按钮）
3. `soft-tip`：关键提示区（当前筛选、状态说明）
4. 内容区：统计卡、图表、表格、表单

推荐顺序：

- 页头
- 提示
- 统计/筛选
- 主数据表或图表
- 分页与操作弹窗

## 3. 文案规范

1. 标题：聚焦动作对象（如“系统公告管理”）
2. 副标题：说明业务价值与使用方式（1 句话）
3. 空状态：
   - 标题：说明“当前无数据”
   - 描述：给出下一步动作建议

## 4. 交互规范

1. 刷新按钮统一放在页头操作区。
2. 筛选条件变化时，默认重置到第一页。
3. 批量操作必须二次确认。
4. 提交成功/失败必须有消息反馈。

## 5. 视觉规范

1. 主按钮：渐变主色（已由全局 `el-button--primary` 统一）。
2. 卡片：玻璃态背景 + 圆角 + 阴影。
3. 表格空状态：统一使用 `empty-state` 视觉块。
4. 页面过渡：可使用 `fade-in-page` 动效类。

## 6. 响应式规范

1. `page-header` 在窄屏下纵向排列。
2. 操作区按钮与筛选控件支持自动换行。
3. 图表容器保持最小高度，避免塌陷。

## 7. 加载与空态规范

1. 首屏关键统计推荐使用骨架屏（`skeleton-grid` / `skeleton-card`）。
2. 表格统一使用 `#empty` 插槽承载 `empty-state` 视觉块。
3. 图表加载阶段建议开启 `v-loading`，避免空白闪烁。

## 8. 认证页品牌规范

1. 登录/注册页统一使用 `auth-brand` 区块展示品牌价值。
2. 品牌区包含：标题、说明文案、3 个能力标签。
3. 移动端自动隐藏品牌区，优先保证表单可用性。

## 9. 推荐复用样式类

- `page-shell`
- `page-header`
- `page-title`
- `page-subtitle`
- `page-actions`
- `soft-tip`
- `fade-in-page`
- `empty-state`
- `empty-illustration`
- `empty-title`
- `empty-desc`
- `skeleton-grid`
- `skeleton-card`
- `auth-brand`
- `auth-brand-title`
- `auth-brand-desc`
- `auth-brand-tags`
- `auth-brand-tag`

以上类均定义在：`frontend/src/styles/theme.css`。

## 10. 图表语义配色规范

1. 风险状态色固定为：高风险 `#d64545`、中风险 `#e38b2c`、低风险 `#2f7d32`。
2. 预警状态色固定为：未处理（OPEN）使用高风险色，已处理（CLOSED）使用低风险色。
3. 常规图表调色盘统一来自 `frontend/src/constants/chart-theme.js`，避免页面各自定义颜色造成认知负担。
4. 折线/柱状图纵轴建议统一启用虚线网格（`CHART_SPLIT_LINE`）。

## 11. 首次进入引导规范

1. 核心复杂页面（监控、洞察、预警工作台、周月报）应提供首次进入提示。
2. 首次引导通过 `showFirstVisitGuide` 实现，使用 localStorage 存储一次性标记。
3. 引导文案聚焦“建议操作顺序”，不解释底层实现。
4. 引导应在页面首屏可交互后触发，避免阻断初始化请求。
